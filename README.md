# Zonemaster Benchmarking Tools

These tools automates the benchmarking of **Zonemaster**, a DNS zone testing and validation framework.
It runs predefined tests on a list of domains, measures performance and consolidates results into a single dataset for analysis.

- `benchcli` is a python script to benchmark `zonemaster-cli` and `gonemaster`
- `zonemaster-benchmark` is a perl script to benchmark `zonemaster-engine`

## ./benchcli usage

```shell
âťŻ ./benchcli zonemaster.se --runs 3
```

options availables:

- `(--batch [BATCH] | --report [REPORT ...] | domain)`: one of these is required
   1. `domain`: run the test on this domain only
   2. `--batch [BATCH]`: run the test on multiple domain, `BATCH` file contain one domain name by line 
   3. `--report [REPORT ...]`: generate the report on previous run, `REPORT` file is file generated by `--output` option
- `[--log-level [{CRITICAL,ERROR,WARNING,INFO,DEBUG}]]`: set log-level
- `[--cache]`: use the `--restore`, if the cache file doesnt exist, it will be generated first
- `[--cache_dir CACHE_DIR]`: set the directory where cache file are stored, by default it is `/tmp`
- `[--runs [RUNS]]`: number of runs by domain, it's passed to `hyperfine`
- `[--output OUTPUT]` : write result into the file `OUTPUT`
- `[--profile PROFILE]` : pass a profile to `zonemaster-cli`, usefull if you want to test a limited set of testcase
- `[--bin [BIN]]` : you can benchmark other tools, like `gonemaster`, if they have the same CLI interface.
